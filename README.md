# AI Layer Factory

Учебный исследовательский проект по ручной реализации нейросети на `NumPy`.

Этот ноутбук  сфокусирован не на использовании готовых high-level библиотек а на том чтобы **своими руками собрать архитектуру нейросети из отдельных слоёв**, понять прямой проход, обратное распространение ошибки, pooling, dropout и базовую организацию обучения.

Проект представляет собой мини-шаблон для конструирования собственных сетей: можно задавать конфигурации слоёв, собирать сеть из свёрточных и полносвязных блоков и запускать обучение на подготовленном датасете.

## Что здесь реализовано
- Полносвязный слой `Direct_Layer`.
- Свёрточный слой `CNN_Layer`.
- Max-pooling внутри свёрточного слоя.
- Dropout-регуляризация.
- Функции активации: `ReLU`, `tanh`, `sigmoid`, `softmax`.
- Прямой проход по сети.
- Обратное распространение ошибки.
- Класс `Teacher` для обучения сети.
- Класс `Tester` для проверки результатов на тестовом наборе.
- Графики ошибок обучения через `matplotlib`.

## Идея проекта
Цель проекта в том чтобы на практике понять:
- как хранится структура сети;
- как согласуются размерности между слоями;
- как устроен forward pass;
- как считаются градиенты;
- как устроено обучение на батчах;
- почему ошибки в геометрии слоёв могут полностью ломать обучение.

## Датасет
В ноутбуке используется вручную собранный датасет изображений цифр в виде бинарных матриц `15x15`.

Есть:
- обучающая выборка;
- тестовая выборка;
- one-hot векторы правильных ответов.

Внутри ноутбука доступны разные варианты порядка обучающих данных:
- `learnSetHorizontalSorted`
- `learnSetVerticalSorted`
- `learnSetUnsorted`

По умолчанию используется:

```python
learn_dataset = learnSetHorizontalSorted
```

## Текущая конфигурация сети
Сейчас в ноутбуке по умолчанию выставлена архитектура с **тремя свёрточными слоями**:
- первый `CNN`;
- второй `CNN`;
- третий `CNN + pooling`;
- далее полносвязный классификатор;
- выходной слой `softmax`.

Это сделано специально, чтобы можно было сразу проверять корректную сборку нескольких свёрточных слоёв подряд.

## Что было исправлено
В проекте была критическая логическая ошибка в сборке свёрточных слоёв.

Проблема была в том, что выход `CNN`-слоя раньше хранился в искажённой форме: **ширина карты признаков смешивалась с количеством фильтров**. Из-за этого при сборке цепочки `CNN -> CNN` следующий слой получал не корректный набор feature maps, а фактически сломанный двумерный тензор.

Дополнительно были исправлены:
- производная `sigmoid`;
- градиент выходного `softmax`;
- логика backprop через pooling и stacked CNN;
- масштабирование обновления весов в обучении;
- лишние импорты, не нужные для запуска ноутбука.

## Как запустить
1. Открой `AILearning_NeuralPattern_ACTUAL.ipynb`.
2. Выбери Python-интерпретатор, где установлены `numpy` и `matplotlib`.
3. Сделай `Restart Kernel`.
4. Запусти ноутбук сверху вниз.

Минимально важные ячейки:
- окружение (`#Environment`);
- формирование датасета;
- функции и классы;
- конфигурация `neural_net`;
- `#  Learning process:`;
- `# Neural network testing:`.

- итоговое предсказание после обучения;

## Для кого этот проект
Проект особенно полезен, если хочется руками понять:
- как работает свёрточная сеть;
- как устроена передача размерностей между слоями;
- как строится простая обучающая обвязка без готового фреймворка;
- какие классы и абстракции нужны для собственного мини-движка нейросетей.

## Ограничения
- Это учебный ноутбук, а не production-ready framework.
- Вся логика живёт в одном `.ipynb`, а не разнесена по модулям.
- Реализация ориентирована на понимание процесса, а не на максимальную производительность.
- Датасет маленький и демонстрационный.

## Почему проект всё ещё интересный
Несмотря на учебный формат, здесь интересно именно:
- ручная реализация ключевых частей нейросети;
- попытка сделать не одну жёстко прошитую сеть а именно **шаблон для сборки разных архитектур**;
- аккуратная идея инкапсулировать функциональность по слоям;
- фокус именно на понимании механики фундаментальных основ работы нейросетей.
